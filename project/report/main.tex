\section{What we should write}

\begin{itemize}
    \item The project title, the group number, the names of all group members, and a statement of the fact that this is the project for this course this year.
    \item A table of contents,
    \item An introduction, containing the motivation for the project and the context it should be used in, as well as a summary of what the program does.
    \item Use cases: a guide for how to actually use your program, including key examples.
    \item Program documentation: a description of how your program really works, including at least:
        \begin{itemize}
            \item The overall design of your program: which are its constituent parts, and how do they interact? 
            \item Description of important data structures and algorithms, including the problem that they solve and their interface.
            \item Description of the major functions of the program, with functional specifications. Describe the control flow (i.e., how functions call each other), and the use of helper functions.
            \item Brief description of tests and their coverage.
        \end{itemize}
    \item Discussion
        \begin{itemize}
            \item Description of technical decisions/choices that you made during the project, together with your reasoning for making a particular choice.
            \item Strengths and weaknesses of the program. You may have created a simple design, written a clear interface, implemented an interesting algorithm or data structure, or achieved high test coverage. There may be parts of your program that work but not as well as you would like, or functionality that despite valiant attempts you have not succeeded in implementing properly.
        \end{itemize}
    \item A short conclusion.
\end{itemize}
